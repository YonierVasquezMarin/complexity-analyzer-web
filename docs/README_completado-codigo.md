# Completado de Código

## ¿Qué es el completado de código?

El completado de código es una funcionalidad que permite completar automáticamente pseudocódigo que está incompleto o que tiene instrucciones pendientes de completar.

## ¿Cuándo se usa?

Esta funcionalidad es útil cuando:
- El código tiene secciones incompletas marcadas con instrucciones especiales
- Hay partes del código que necesitan ser expandidas o completadas
- Se quiere generar código adicional basado en el código existente

## ¿Cómo funciona?

### Proceso de Completado
1. El usuario tiene código con instrucciones de completado (por ejemplo, "Completar: poner otro bucle pero con la letra j")
2. El sistema envía el código al servidor
3. El servidor procesa el código y genera la versión completada
4. El código completado reemplaza o actualiza el código original

### Resultado
Después del completado, el código queda en un formato más completo y listo para ser analizado. Las instrucciones de completado se resuelven y se convierten en código real.

## Integración con el Flujo de Trabajo

El completado de código es un paso opcional en el proceso de análisis:
1. Escribir o cargar código (puede estar incompleto)
2. **Completar código** (opcional)
3. Convertir el pseudocódigo
4. Analizar la complejidad

## Ventajas

- **Ahorro de tiempo**: No es necesario escribir manualmente código repetitivo
- **Consistencia**: El código completado sigue patrones consistentes
- **Facilidad**: Permite trabajar con código incompleto y completarlo cuando sea necesario

## Limitaciones

- Requiere conexión al servidor para funcionar
- El completado depende de la calidad del código original y las instrucciones proporcionadas
- No siempre puede completar código muy complejo o ambiguo
