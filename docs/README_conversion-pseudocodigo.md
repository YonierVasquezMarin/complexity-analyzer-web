# Conversión de Pseudocódigo

## ¿Qué es la conversión de pseudocódigo?

La conversión de pseudocódigo es el proceso mediante el cual el código escrito por el usuario se transforma en un formato estándar y normalizado que puede ser analizado correctamente por el sistema.

## ¿Por qué es necesaria?

El pseudocódigo puede escribirse de diferentes maneras, con variaciones en:
- La sintaxis
- Las palabras clave utilizadas
- El formato
- Las instrucciones especiales

La conversión estandariza todo esto para que el sistema pueda analizarlo de manera consistente.

## ¿Qué hace la conversión?

### Normalización
Convierte diferentes formas de escribir lo mismo en un formato único. Por ejemplo, diferentes formas de escribir un bucle se convierten en la misma estructura estándar.

### Resolución de Instrucciones
Si el código tiene instrucciones de completado (como "Completar: poner otro bucle"), estas se resuelven y se convierten en código real.

### Limpieza
Elimina o corrige inconsistencias en el formato que podrían causar problemas en el análisis.

## Visualización de la Conversión

### Vista Antes y Después
La plataforma ofrece una vista especial donde se puede ver:
- **Antes**: El pseudocódigo original tal como lo escribió el usuario
- **Después**: El pseudocódigo convertido en formato estándar

Esta comparación lado a lado permite entender qué cambios se realizaron durante la conversión.

## ¿Cuándo se convierte?

La conversión ocurre cuando:
- El usuario solicita convertir el código explícitamente
- Se necesita el código convertido para realizar un análisis
- El sistema requiere el formato estándar para procesar el código

## Resultado de la Conversión

Después de la conversión:
- El código original se mantiene intacto
- Se crea una versión convertida que se guarda junto al archivo
- El código convertido está listo para ser analizado
- Ambos (original y convertido) están disponibles para visualización

## Importancia para el Análisis

La conversión es fundamental porque:
- Garantiza que el análisis se realice sobre código válido y consistente
- Permite que el sistema entienda correctamente la estructura del algoritmo
- Facilita la detección de patrones y estructuras de control
- Asegura resultados de análisis precisos y confiables
